# Skill: SpringBoot Migration Plan

## Purpose
Define the target version and an incremental migration plan.

## Inputs
- Migration Baseline artifact
- Target Spring Boot version decision
- Compatibility notes or release guidance

## Steps
1. Confirm the single target Spring Boot version and Java baseline.
2. Identify compatibility gaps and required code or config changes.
3. Draft an incremental plan with small, reversible steps.
4. Populate the Change Plan template.

## Outputs
- Change Plan artifact using `ai/templates/change_plan.md`.
- Compatibility Gap List artifact describing required changes.
