# Skill: SpringBoot Migration Assessment

## Purpose
Establish the baseline state of the Spring Boot service before migration.

## Inputs
- Current build files (Maven or Gradle)
- Current runtime environment details
- Configuration files
- Dependency list

## Steps
1. Record current Spring Boot, Java, and build tool versions.
2. Inventory dependencies and identify overrides or custom versions.
3. List configuration files and note any custom properties or profiles.
4. Capture baseline startup or test command if available.

## Outputs
- Migration Baseline artifact with versions, dependency inventory, and config inventory.
