# Workflow: SpringBoot Migration

## Required Sequence
1. Assess current state: versions, dependencies, configuration, and build setup. Use `ai/skills/07_springboot_migration_assess.md`.
2. Define target version and migration plan. Use `ai/skills/08_springboot_migration_plan.md`.
3. Execute incremental migration changes with minimal diffs. Use `ai/skills/09_springboot_migration_execute.md`.
4. Verify startup, tests, and compatibility. Use `ai/skills/10_springboot_migration_verify.md`.
5. Produce a PR-ready summary. Use `ai/skills/06_pr_ready_summary.md`.

## Required Artifacts
- Change Plan
- PR Summary

## Stop Conditions
- If baseline versions or dependencies cannot be identified, stop and request clarification.
- If target version is not confirmed, stop and request a decision.
