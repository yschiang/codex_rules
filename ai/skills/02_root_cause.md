# Skill: Root Cause Analysis

## Purpose
Pinpoint the exact file, function, and condition that causes the issue or defines system constraints.

## Inputs
- Repro Notes or failing test
- Logs or stack traces
- Relevant code paths

## Steps
1. Trace the execution from entry point to failure.
2. Identify the precise condition that triggers the issue.
3. Validate by toggling the condition or adding a targeted check.

## Outputs
- Root Cause Report artifact with file, function, condition, and evidence.
