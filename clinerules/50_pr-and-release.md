# PR And Release

## PR Content Checklist
- Clear summary of intent and scope.
- Root cause or design intent documented.
- Tests listed with commands and results.
- Risks and rollback plan included.

## Risk And Rollback
- Identify blast radius and mitigation.
- Provide a rollback path or feature flag plan.

## Commit Hygiene
- Keep commits small and topic-focused.
- Avoid mixing formatting or refactor with functional changes.
- Use descriptive commit messages.
