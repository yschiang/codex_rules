# Skill: Write Failing Test

## Purpose
Create a red test that captures the bug or verifies feature acceptance criteria.

## Inputs
- Repro Notes or Acceptance Criteria
- Root Cause Report if available
- Existing test patterns

## Steps
1. Select the lowest-level test that proves the behavior.
2. Write the test to express the expected outcome.
3. Run the test and confirm it fails.

## Outputs
- Failing Test artifact with file path and test name.
- Red Test Evidence with command and failure output summary.
